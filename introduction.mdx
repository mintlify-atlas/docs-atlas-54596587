## What is pkmn/engine?

`pkmn/engine` is a high-performance Pokémon battle simulation engine written in [Zig](https://ziglang.org/), with first-class TypeScript/JavaScript bindings and a C API for use in other languages. It is designed for tooling, embedded systems, and artificial intelligence use cases where raw simulation speed is critical.

The engine is **more than 1000× faster** than [Pokémon Showdown](https://pokemonshowdown.com/) when playing out supported formats in compatibility mode. It is frame-accurate and [bug-for-bug compatible](http://www.catb.org/jargon/html/B/bug-for-bug-compatible.html) with both the original Pokémon game cartridges and the Pokémon Showdown simulator.

<Warning>
  This project is under active development. The `main` branch contains breaking changes that may not be fully documented. Wait for the forthcoming v0.1 release before depending on this project in production.
</Warning>

## How it works

The engine exposes a minimal API centered on two operations:

1. **`update(c1, c2)`** — Advance the battle state given both players' choices. Returns a `Result` indicating whether the battle ended and which types of choices are valid next.
2. **`choices(player, result)`** — Enumerate all valid choices for a player at the current decision point.

This design allows the engine to be driven entirely by code — there is no streaming, no async I/O, and no string-based protocol overhead in the hot path.

## What the engine is not

The pkmn engine is a **low-level library**, not a fully featured simulator. The following are explicitly out of scope:

- Team or set validation
- Custom rule ("format") enforcement
- Battle variants other than single (full) or double battles
- User-facing game socket servers or input validation

These concerns are left to higher-level code built on top of the engine.
